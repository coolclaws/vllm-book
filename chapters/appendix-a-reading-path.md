# 附录 A：推荐阅读路径

本书共 22 章加 3 个附录，涵盖了 vLLM 从用户接口到底层内核的完整源码分析。不同读者的知识背景和阅读目标各异，为此我们设计了四条推荐阅读路径，帮助你高效地获取所需知识。

---

## 路径一：快速上手路径

**适合读者**：希望快速了解 vLLM 的核心概念并投入使用的工程师、算法工程师和技术决策者。

**章节序列**：第 1 章 → 第 2 章 → 第 9 章 → 第 19 章

| 章节 | 内容 | 你将学到 |
|------|------|----------|
| 第 1 章：项目概览与设计哲学 | vLLM 的诞生背景、PagedAttention 核心思想、设计原则 | 理解 vLLM 为何存在以及它解决了什么问题 |
| 第 2 章：仓库结构与构建系统 | 代码目录组织、关键文件入口、构建与安装 | 能够从源码构建 vLLM，知道在哪里找到想看的代码 |
| 第 9 章：API 与服务化 | OpenAI 兼容 API、离线推理接口、服务端部署 | 掌握 vLLM 的使用方式，能够搭建推理服务 |
| 第 19 章：多模态模型支持 | 视觉语言模型、图像/视频输入处理 | 了解 vLLM 对多模态场景的支持能力 |

**预计阅读时间**：2-3 小时

**阅读建议**：这条路径注重"是什么"和"怎么用"，跳过了大部分底层实现细节。如果在阅读过程中对某个主题产生了深入了解的兴趣，可以随时切换到核心原理路径的对应章节。

---

## 路径二：核心原理路径

**适合读者**：希望深入理解 PagedAttention 机制、调度算法和 attention 计算原理的系统工程师和研究者。

**章节序列**：第 3 章 → 第 4 章 → 第 5 章 → 第 6 章 → 第 7 章 → 第 13 章

| 章节 | 内容 | 你将学到 |
|------|------|----------|
| 第 3 章：请求的生命周期 | 一个请求从提交到返回结果的完整流程 | 建立对整个系统数据流的全局认知 |
| 第 4 章：Scheduler 与请求调度 | 调度策略、连续批处理、抢占机制 | 理解 vLLM 如何高效地编排并发请求 |
| 第 5 章：Block Manager 与内存管理 | 物理块分配、block table 映射、Copy-on-Write | 掌握 PagedAttention 的内存管理核心 |
| 第 6 章：KV Cache 物理布局 | KV Cache 的 GPU 内存组织、block 结构 | 理解数据在 GPU 上的实际存储方式 |
| 第 7 章：Attention 计算内核 | Flash Attention、PagedAttention kernel 实现 | 深入理解 attention 的高性能计算实现 |
| 第 13 章：Chunked Prefill | 分块预填充的动机、实现与调度集成 | 理解如何平衡 prefill 与 decode 的延迟 |

**预计阅读时间**：6-8 小时

**阅读建议**：这条路径是全书最核心的技术主线。建议结合源码阅读——每章提到的关键文件路径都值得打开对照阅读。第 5 章和第 7 章是理解上的两个"高峰"，可能需要反复阅读和思考。建议先通读第 3 章建立全局视角，再逐章深入。

---

## 路径三：分布式推理路径

**适合读者**：负责多 GPU / 多节点部署的基础设施工程师和 MLOps 工程师。

**章节序列**：第 10 章 → 第 16 章 → 第 17 章 → 第 18 章

| 章节 | 内容 | 你将学到 |
|------|------|----------|
| 第 10 章：Worker 与 Executor 架构 | Worker 抽象、Executor 实现、进程模型 | 理解 vLLM 的分布式执行框架 |
| 第 16 章：Tensor Parallelism | 张量并行的切分策略、AllReduce 通信 | 掌握单机多卡的并行推理机制 |
| 第 17 章：Pipeline Parallelism | 流水线并行、stage 划分、micro-batch 调度 | 理解跨节点的流水线推理方案 |
| 第 18 章：分布式通信 | NCCL 集成、通信组管理、自定义 AllReduce | 深入理解分布式通信的实现细节 |

**预计阅读时间**：4-5 小时

**阅读建议**：阅读此路径前，建议先完成第 1 章和第 3 章以建立基础认知。如果你已经熟悉分布式深度学习的基本概念（如 AllReduce、NCCL），可以直接进入第 10 章。否则建议先阅读第 18 章的基础概念部分。在实际部署中，Tensor Parallelism（第 16 章）是最常用的并行方式，建议重点阅读。

---

## 路径四：全书通读路径

**适合读者**：希望系统、全面地掌握 vLLM 源码的深度学习系统研究者和高级工程师。

**章节序列**：第 1 章 → 第 2 章 → … → 第 22 章 → 附录 A/B/C

| 阶段 | 章节范围 | 主题 |
|------|----------|------|
| 基础认知 | 第 1-3 章 | 项目概览、代码结构、请求生命周期 |
| 核心机制 | 第 4-8 章 | 调度、内存管理、KV Cache、Attention、采样 |
| 用户接口 | 第 9 章 | API 与服务化 |
| 执行架构 | 第 10-12 章 | Worker/Executor、模型加载、计算图执行 |
| 高级特性 | 第 13-15 章 | Chunked Prefill、Speculative Decoding、LoRA |
| 分布式 | 第 16-18 章 | TP、PP、分布式通信 |
| 扩展能力 | 第 19-20 章 | 多模态、自定义模型集成 |
| 优化部署 | 第 21-22 章 | 量化支持、性能调优 |
| 参考资料 | 附录 A/B/C | 阅读路径、类型速查、名词解释 |

**预计阅读时间**：20-30 小时（建议分 2-3 周完成）

**阅读建议**：全书通读是收益最大的方式，但也最耗时。建议采用"两遍法"：第一遍快速浏览，重点关注每章的架构图和小结，建立整体认知；第二遍结合源码深入阅读重点章节。附录 B（核心类型速查）和附录 C（名词解释）可以在阅读过程中随时查阅，不必专门阅读。

---

## 各路径之间的关系

四条路径并非互斥，而是层层递进的关系：

```
快速上手路径 (4 章)
    └──→ 核心原理路径 (6 章)
            ├──→ 分布式推理路径 (4 章)
            └──→ 全书通读路径 (22 章 + 附录)
```

无论选择哪条路径，第 1 章（项目概览）和第 3 章（请求的生命周期）都是推荐的起点。它们为后续所有章节提供了必要的背景知识和全局视角。
